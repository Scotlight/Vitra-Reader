# 主页面虚拟网格需求冻结（2026-03-22）

## 目标

为主页面书籍列表添加虚拟化支持，降低大量藏书场景下的主线程压力、首屏渲染量与滚动卡顿。

## 交付范围

- 仅覆盖主页面书籍网格。
- 保持现有主页面筛选、排序、右键菜单、打开书籍、进度展示、封面懒加载等行为不变。
- 保持书架分组卡片、批注列表、侧边栏、设置面板不进入本轮虚拟化范围。

## 约束

- 不新增第三方虚拟列表依赖。
- 虚拟网格的行高不能写死，必须从当前卡片实际渲染尺寸量测得到。
- 视口尺寸变化、列数变化时，量测结果必须能自动更新。
- 代码应尽量局部改动，优先落在 `src/components/Library/`。

## 验收标准

- 大量书籍时，DOM 中只保留视区附近卡片，而不是一次性渲染全部卡片。
- 改变窗口宽度后，列数与可视窗口计算保持正确。
- 卡片交互不回退：点击打开、右键菜单、进度显示、封面显示仍正常。
- 非网格区域表现不变。

## 非目标

- 不处理书架分组卡片虚拟化。
- 不处理高亮/笔记列表虚拟化。
- 不改动书籍分组、排序能力本身。
