# 2026-03-22 Dead Code Cleanup

## Goal

在不误伤工作树中其他未确认改动的前提下，先将当前 Reader 重构成果以最小安全范围推送到远端，然后继续审计并清理仓库中的死代码。

## Deliverables

- 一条仅包含本轮 Reader 重构与配套审计产物的提交，并已推送到 `origin/main`
- 一份死代码审计结论，明确哪些项被确认、哪些项被跳过
- 一组聚焦死代码清理的最小代码修改

## Constraints

- 不得顺手提交或推送与本轮无关的脏改动
- 保持 TypeScript 严格模式可通过验证
- 修改应优先集中在 Reader 层和明确可证实的死代码项

## Acceptance Criteria

- Reader 拆分成果能够以独立提交推送成功
- 死代码审计具有证据支撑，而非主观判断
- 本轮清理项完成后具备验证命令与结果记录

## Non-goals

- 不做全仓库历史技术债大扫除
- 不处理与本轮目标无关的行为性缺陷

## Assumptions

- 推送范围按最小安全提交执行
- 死代码包括未使用符号、无引用模块、抽离后遗留旧逻辑和明显不可达代码
