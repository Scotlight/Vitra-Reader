# 项目结构分析需求冻结

- 日期：`2026-03-21`
- 主题：`当前项目结构分析与优先阅读文件`
- 模式：`benchmark_autonomous`

## Goal

基于当前仓库状态，分析项目的目录职责、启动入口、主数据流与阅读链路，并产出一份适合首次接手时的优先阅读清单。

## Deliverable

输出一份中文说明，至少包含：

1. 顶层目录职责概览
2. Electron → React → Library/Reader 的入口关系
3. 导入书籍 → 解析元数据 → 打开阅读器 的主链路
4. 首批建议阅读的关键文件及理由

## Constraints

- 只基于本地仓库证据，不猜测未看到的实现
- 区分“事实”和“推断”
- 不改动业务代码
- 保持结论可回溯到具体文件

## Acceptance Criteria

- 明确顶层目录分层
- 明确主进程与渲染进程入口
- 明确 Zustand / Dexie / Vitra Engine 的落点
- 给出 6–10 个最值得先看的文件

## Non-Goals

- 不做功能修复
- 不做架构重构
- 不做测试变更
