# 主页分组混排与排序需求冻结（2026-03-22）

## 目标

将原“书架”能力重命名为“分组”，并把主页根视图改为“分组卡片 + 未分组图书”混排视图，支持右键空白处新建分组，以及长按排序。

## 交付范围

- 用户可见文案统一从“书架”改为“分组”。
- 主页面根视图支持分组卡片与未分组图书混排。
- 主页面空白区域右键支持“新建分组”。
- 主页混排顺序与分组内图书顺序支持长按拖动排序。
- 兼容旧 `shelves` / `shelfBookMap` 数据并软迁移到新状态键。

## 约束

- 保持现有图书右键菜单、打开图书、批注列表、收藏/高亮/回收逻辑不回退。
- 主页关键字搜索时退回纯图书结果，不混入分组卡片。
- 不在本轮删除旧状态键，只做兼容读取与新键写入。
- 代码改动尽量局部，主要集中在 `src/components/Library/` 与 `src/hooks/`。

## 验收标准

- 首页根视图不再把分组单独放一块，而是与未分组图书共用同一网格。
- 右键主页空白区域可直接新建分组。
- 主页长按项目可调整混排顺序；进入分组后长按图书可调整该分组内部顺序。
- 老用户已有书架数据可无感迁移到分组数据结构。
- `npm run build --silent` 与新增定向测试通过。

## 非目标

- 本轮不处理收藏/笔记/高亮/回收页的混排排序。
- 本轮不实现分组卡片的专属右键菜单。
- 本轮不物理删除旧 `shelves` 配置键。
