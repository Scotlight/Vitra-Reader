# Library Focus Cover Flicker And Open-Book CPU

## 背景

当前主页面存在两个未收敛问题：

1. 应用失去焦点后返回主页面时，书籍封面会先短暂显示格式占位图，再恢复为真实封面。
2. 打开书籍时仍会出现明显 CPU 峰值，前一轮优化未达到预期。

从当前实现看，主页面封面并未随书籍列表一并保存在 Zustand 中，而是由 `BookGrid` 在渲染时逐本从 Dexie 按需读取。该设计有利于控制内存，但若主页面在焦点恢复时触发整块重建，就会重新经过“占位图 → 异步取封面 → 真实封面”的流程。

## 目标

1. 消除主页面在焦点恢复后的封面回退闪烁。
2. 定位打开书籍时的 CPU 热点，并针对根因做收敛，不做凭感觉的补丁。
3. 保持现有库页、阅读器和 Provider 对外行为兼容。

## 验收标准

- 主页面停留期间，切出应用再切回，已显示过的封面不再回退为默认占位图。
- 修复不依赖扩大内存常驻量来掩盖问题。
- 对打开书籍 CPU 问题给出明确热点定位，并在代码中落实至少一处根因修复。
- `npm run build --silent` 通过。

## 非目标

- 本轮不改书籍元数据存储模型。
- 本轮不做新的大规模阅读器架构重写。
- 本轮不把所有封面重新塞回 Zustand 常驻状态。
