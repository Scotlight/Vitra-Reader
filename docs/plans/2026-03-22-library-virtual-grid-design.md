# 主页面虚拟网格设计（2026-03-22）

## 方案选择

采用无额外依赖的轻量虚拟网格方案。

## 设计要点

1. `LibraryView` 继续持有滚动容器，只把滚动元素引用传给书籍网格。
2. `BookGrid` 保留书架分组卡片的普通渲染，只把书籍卡片区域切换为虚拟网格。
3. 虚拟网格先渲染少量探测卡片，量测真实列数、行间距、卡片高度，再切换为窗口化渲染。
4. 行高不写死，改为缓存可见行的实际高度；未知行使用当前量测平均值估算，总高度随量测结果迭代修正。
5. 当容器宽度变化时，重置列布局并重新进入探测量测阶段，保证响应式网格与实际 CSS 尺寸一致。

## 关键取舍

- 不引入 `react-window`，减少后续维护和样式适配成本。
- 不依赖固定卡片高度，避免未来卡片样式调整后要回头改虚拟化常量。
- 对小列表保持低复杂度：即使启用同一套组件，也优先保证稳定行为。
