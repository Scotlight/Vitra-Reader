# 主页分组混排与排序设计（2026-03-22）

## 方案选择

采用“状态层先迁移、视图层再混排、交互层最后补长按排序”的局部重构方案。

## 设计要点

1. 用 `groups`、`groupBookMap`、`groupBookOrder`、`homeOrder` 替代旧书架命名，并保留旧键读取能力。
2. 首页根视图只在 `activeNav === 'all' && !activeGroupId && keyword === ''` 时进入混排模式。
3. 混排项目由“全部分组卡片 + 未分组图书”组成，顺序由 `homeOrder` 决定。
4. 进入分组后只显示该分组内图书，顺序由 `groupBookOrder` 决定，不再受普通排序按钮影响。
5. 右键空白菜单只在主页根视图启用，点击卡片或图书时不触发。
6. 长按排序采用轻量实现：`setTimeout` 激活拖拽，`document.elementFromPoint()` 查找目标项，释放时持久化顺序。

## 关键取舍

- 搜索态不混入分组卡片，避免重复项目与复杂的筛选规则。
- 不引入新的拖拽库，降低维护负担并保持虚拟网格现有实现可复用。
- 不删除旧状态键，降低迁移风险，后续可再做清理批次。
