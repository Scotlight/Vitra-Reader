# ADR-0002：暂时禁用 PDF text layer

## 状态

已采用

## 背景

PDF 页面理论上可在像素层之上叠加透明文本层，以支持文本选择、复制和更精细的语义交互。但该能力会带来明显的性能与稳定性负担。

## 决策

当前默认禁用 PDF text layer，仅保留：

- 像素层
- 链接交互层

## 原因

- 文本层生成与对齐成本高
- 会增加页面渲染压力
- 对滚动流畅度、首屏与内存有负面影响
- 当前优先保证可读性、稳定性与链接交互

## 影响

正向：

- 减少单页渲染成本
- 降低内存与布局压力
- 提升大 PDF 的可用性

代价：

- 无法完整支持 PDF 文本选择/复制体验
- 后续若重启该能力，需要重新解决对齐与性能问题

## 后续约束

- 任何恢复 text layer 的尝试都必须附带性能回归验证
- 若恢复，需新增独立 ADR 记录新策略
