# 分页虚拟化稳定性修复需求

日期：2026-05-01

## 目标

解决上一轮虚拟化验收中的两个阻塞项：

1. 分页快速连续跳转时，旧章节异步加载结果不得覆盖新章节。
2. 大页数分页章节的页窗外内容不能只做 `visibility: hidden`，需要释放页窗外可渲染块的子 DOM 与媒体资源，并能在回到页窗时恢复。

## 约束

- 保持 `PaginatedReaderView` 外部调用方不变。
- 不重写分页排版模型，不破坏 CSS columns 的页码计算。
- 保留布局壳，避免页码漂移、目录跳转、选区坐标失效。
- 小章节仍不启用水平页窗处理。
- 修改后必须补充定向测试并执行验证。

## 验收标准

- 快速连续跳转测试能证明旧任务结果被丢弃。
- 页窗外元素进入脱水状态时，子 DOM 被移除，媒体 `src/srcset/poster` 被清理。
- 页窗内元素恢复后，原始内容和媒体属性恢复。
- 空白页检测仍识别页窗隐藏元素，不把页窗隐藏当真实空白。
- `vitest` 定向测试、`npm run lint`、`npm run build --silent` 通过。
